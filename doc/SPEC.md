# 設計仕様 (SPEC)

このドキュメントは開発者向けの設計思想、アーキテクチャ、データモデル、ロードマップをまとめています。

## 概要

Rust + SQLite を基盤とした軽量 CMS。テンプレートに「更新可能領域」を自由に定義し、管理画面からウィジェットとして直接編集できるのが最大の特徴です。企業サイトの動的ブロック（カルーセル・お知らせ・特集など）を柔軟に運用できます。

## アーキテクチャのポイント

- 単一バイナリで配布・運用が容易
- データ永続化は SQLite のみ（外部 DB 不要）
- 公開サイトはテンプレートエンジンで動的生成
- 管理画面はコンパイル時テンプレート検証で安全性を確保
- セッションは署名付き Cookie で管理

## 主なデータモデル

- レイアウト：共通枠（shell）と静的ファイル
- ページ：URL と本文テンプレート
- 投稿：プレースホルダー配下のコンテンツ（お知らせなど）
- ウィジェット：HTML テンプレートと設定スキーマ
- メディア：画像・ファイルの管理
- オプション：サイト名などの基本設定

## ロードマップ（主な未対応機能）

- 細かい権限制御
- カテゴリ・タグ
- メニュー・RSS・予約公開
- EC 機能（商品・注文）

詳細な実装状況や技術的な設計はソースコードとテストを参照してください。

## テスト仕様

本プロジェクトは `tests/` 配下の統合テストを中心に、主要機能の回帰防止とエッジケースの検証を行っています。主なテストカテゴリは以下の通りです。

### 認証・セッション

- 管理画面未認証アクセス時の `/admin/login` へのリダイレクト
- 誤パスワードでのログインエラー表示
- 正しい認証情報での管理画面アクセス許可
- ログアウト後の再ログイン要求
- API セッションでのログイン/ログアウトと Cookie 認証の連携

### 投稿・ゴミ箱機能

- 投稿のゴミ箱移動後、一覧に表示されること
- ゴミ箱からの復元（公開/非公開状態の復元）
- 完全削除（purge）によるデータ完全除去
- ゴミ箱外の投稿に対する復元/削除の拒否
- プレースホルダー削除時の「アクティブ投稿存在時ブロック」チェック

### ページ・レイアウト管理

- ページ作成時のファイル永続化と削除時のファイル除去
- レイアウトのエクスポート（マニフェスト・shell・ページを含む）
- インポート時の新規作成・上書き・スキップ・リネーム処理
- URL パス衝突の検出と回避（接頭辞付与）

### ウィジェット管理

- ウィジェット HTML のレンダリング（固定テンプレート変数含む）
- プレビュー編集モードでのマーカー付与
- カスタムウィジェットの作成・エクスポート/インポートラウンドトリップ
- プレースホルダー存在時の削除ブロック

### データベース管理画面

- CMS テーブルの一覧表示
- ユーザー定義テーブルの作成・カラム編集・型変更制約
- データ行の一覧・ページネーション・API 取得
- システムテーブルへの書き込み拒否と通知表示
- シード機能によるテストデータ生成

### API・バックアップ

- 未認証 API への 401 返却と認証済み CRUD の基本動作
- バックアップ ZIP のエクスポート（マニフェスト・DB・レイアウトファイル）
- インポート時のマニフェスト検証・安全なパスチェック・バージョン互換性

### その他の機能

- コンタクトフォームの二重送信防止・投稿作成・成功/エラー表示
- ファビコン登録の制限と公開ページへの反映
- 静的ファイルの同期・アップロード・配信
- プレビュー編集モード UI の表示確認
