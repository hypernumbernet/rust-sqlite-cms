# rust-sqlite-cms

Rust + SQLite で構築された高性能 CMS です。1000万件規模のデータを軽快に扱える強力な DB 統合が最大の特徴で、ホームページのお手軽な編集ニーズに応えるだけでなく、経理業務や各種事務所のデータ管理にもそのまま活用できます。テンプレートの任意領域をウィジェットとして直接編集可能で、画面は用途に合わせて自由にカスタマイズ可能です。

- 単一バイナリで配布・運用しやすい構成
- 組み込み SQLite によるシンプルなデータ永続化
- サーバーサイドレンダリングにより、テンプレートを安全かつ柔軟に管理できます

## 機能

現時点で利用できる主な機能は以下の通りです。

### 公開サイト

- テンプレートエンジンによるサーバーサイドレンダリング
- ウィジェット埋め込みによる動的ブロック（カルーセル・お知らせ・ヒーロー画像など）の柔軟な運用
- 企業ホームページサンプルの提供（`/`）

### ページ管理

- ページの作成・編集・削除・公開状態管理
- レイアウト選択、プリセットテンプレート対応、プレビュー機能

### 投稿・コンテンツ管理

- お知らせなどの投稿作成・編集・公開・ゴミ箱管理
- プレースホルダーによる投稿のグループ化

### ウィジェットシステム

- ウィジェットタイプの定義（HTML構成・設定スキーマの編集）
- プリセットウィジェット（news / image / carousel など）の提供

### メディア・レイアウト管理

- 画像・ファイルのアップロード・一覧・削除
- 共通レイアウト（shell.html）、静的アセット、ページテンプレートの管理

### サイト運用・管理

- サイト名・説明・URL の編集
- 管理ユーザーの作成・編集・削除（既定の `admin` は削除不可）
- SQLite データベースの直接閲覧（テーブル・ビュー・データ）
- 開発用サンプルデータの投入

### API・設定

- REST API（`/api/v1`）による各リソースの JSON 操作（Cookie セッション認証）
- `work/config.toml` および環境変数（`CMS_*`）による柔軟な設定
- 初回起動時の SQLite データベース自動生成とスキーマ適用
- 単一バイナリ配布と組み込み SQLite によるシンプルな運用

### 技術基盤

- **Rust**（Actix Web + Askama）による高速かつ安全な実装
- **SQLite** の強み
  - 単一ファイルで完結。バックアップ・移行が極めて容易
  - 外部データベースサーバー不要。ゼロ設定で即利用可能
  - 1000万件規模のレコードも軽快に扱える高いパフォーマンス
  - ACID トランザクションによる高いデータ信頼性
  - 組み込み型のため、経理・事務所システムのような中小規模用途に最適

未対応の主な機能（今後の予定）:
- ロール/権限の細かい制御
- カテゴリ・タグなどのタクソノミー
- ナビゲーションメニュー・RSS・予約公開
- 商品・注文など EC 機能

## セットアップ

### 前提

- Rust（最新の stable 版を推奨）
- Cargo（Rust のビルドツール）

### ビルド・実行

```bash
git clone <repository-url>
cd rust-sqlite-cms
cargo run
```

開発・動作確認用に、管理ユーザー `admin` のパスワードを常に固定値 `testpass` にするテストモードもあります:

```bash
cargo run -- --test
```

`--test` 指定時は起動のたびに `admin` のパスワードが `testpass` に設定されます。本番環境では使用しないでください。

初回起動時に以下の処理が自動で行われます:

- `work/config.toml` の確認（無ければ `config.example.toml` から生成）
- `data/cms.db` への接続とマイグレーション適用
- 既定の管理者ユーザー `admin` の確認
- `work/` ディレクトリの初期化（レイアウトのシード、uploads ディレクトリ、トップページの確保）

`work/config.toml` が無くてもデフォルト値で起動します。設定は環境変数でも上書きできます（例: `CMS_BIND_ADDR=0.0.0.0:3000 cargo run`）。

## 管理画面

ブラウザで `http://127.0.0.1:3000/` にアクセスすると企業ホームページのサンプルが表示されます。カルーセルやお知らせ欄は、管理画面で公開状態にした投稿やメディアをウィジェットとして動的に生成します。

`http://127.0.0.1:3000/admin` にアクセスすると、未ログイン時はログイン画面へリダイレクトされます。ログイン後、以下の管理が可能です:

- お知らせなどの投稿作成・編集・公開・ゴミ箱管理
- ページの作成・編集・削除とレイアウト選択・プレビュー
- 共通レイアウト（shell.html）や静的アセット・ページテンプレートの管理
- ウィジェットタイプ（HTML構成・設定スキーマ）の定義とプリセット管理
- 画像・ファイルのアップロード・一覧・削除
- 開発用サンプルデータの投入
- サイト名・説明・URL の編集
- 管理ユーザーの作成・編集・削除
- SQLite のテーブル・ビュー・データ直接閲覧

### ログイン

- 管理画面はログイン必須です
- **テストモード**（`cargo run -- --test`）: ログイン名 `admin`、パスワード `testpass`
- **通常起動**（`cargo run`）: 初回起動時、DB に `admin` が無い場合はランダムな初期パスワードが起動ログに一度だけ出力されます
- REST API もログイン必須です（セッション API を除く）

## 設定ファイル

設定は `work/config.toml` に保存します。初回起動時に自動生成されます。サイト名・説明は管理画面の `/admin/settings` から編集でき、データベースと同期されます。

設定の優先順位は **デフォルト値 → `work/config.toml` → 環境変数（`CMS_*`）** です。

主な設定項目:

- `[server]` — リッスンアドレス
- `[database]` — SQLite データベースファイルのパス
- `[paths]` — アップロード先や作業ディレクトリ
- `[site]` — 表示名・説明
- `[session]` — セッション Cookie の設定
- `[security]` — セッション署名用シークレット（本番では必ず設定してください）

本番環境では `security.session_secret` を必ず固定値に設定してください。未設定のまま運用すると、再起動のたびに全セッションが無効になります。

## リバースプロキシを使ったデプロイ

本番環境で公開する場合は、Nginx や Caddy などのリバースプロキシを前段に配置することをおすすめします。

### 利点

- HTTPS（Let's Encrypt など）を簡単に導入できる
- 独自ドメインでのアクセスが可能になる
- 静的ファイルのキャッシュやセキュリティヘッダーの追加がしやすい
- アプリケーションの再起動時にもダウンタイムを最小限に抑えられる

### 基本的な考え方

アプリケーションは `127.0.0.1:3000` で待ち受ける設定のままにし、リバースプロキシが外部からのリクエストをすべて転送する形にします。セッション署名鍵などの本番向け設定は環境変数で渡してください。

### Nginx の設定例

```nginx
server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

HTTPS を有効にする場合は Certbot などで証明書を取得し、443 番ポートの設定を追加してください。

### Caddy の設定例（HTTPS 自動化が簡単）

```
example.com {
    reverse_proxy 127.0.0.1:3000
}
```

Caddy は証明書の取得・更新を自動で行ってくれます。

### Apache の設定例

```apache
<VirtualHost *:80>
    ServerName example.com

    ProxyPreserveHost On
    ProxyPass / http://127.0.0.1:3000/
    ProxyPassReverse / http://127.0.0.1:3000/
</VirtualHost>
```

HTTPS を利用する場合は、Let's Encrypt などで証明書を設定した上で、443 番ポートの VirtualHost を追加してください。mod_proxy モジュールが必要です。

本番運用時は、必ず `CMS_SESSION_SECRET` などの重要な設定を環境変数で指定するようにしてください。

## ページとレイアウト

公開サイトは「レイアウト」と「ページ」の2層で管理します。

**レイアウト**ではサイト全体の共通デザイン（ヘッダー・フッター・ナビゲーションなど）を一括管理できます。shell.html や CSS を編集するだけで全ページに反映されるため、サイト全体の見た目を効率的に統一・更新できます。

**ページ**では各URLごとに独立したコンテンツを作成・編集できます。レイアウトを選択して素早くページを構築し、プリセットテンプレートやプレビュー機能を使って効率的に作業可能です。ウィジェットを配置することでカルーセルやお知らせ欄などの動的ブロックも柔軟に組み込めます。

変更は即時に反映され、再起動は不要です。管理画面のデザインは公開サイトとは完全に分離されているため、安心して編集作業が進められます。
